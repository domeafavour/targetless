# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

| Command | What it does |
|---|---|
| `pnpm dev` | Start Vite dev server on port 3000 |
| `pnpm build` | Build all packages |
| `pnpm test` | Placeholder — no test runner installed (README mentions Vitest but not configured) |
| `pnpm typecheck` | Run `tsc --noEmit` in all packages that have the script |
| `pnpm lint` | Check all files with Biome (linting + formatting) |
| `pnpm format` | Auto-fix all fixable issues with Biome (linting + formatting) |
| `pnpm build:gh-pages` | Build for GitHub Pages deployment |
| `pnpm gen-supabase-types` | Regenerate Supabase TS types from remote schema |

## Structure

```
apps/web/               # React 19 + TanStack Start SPA (Vite, Tailwind v4)
packages/data-access/   # Supabase client + IndexedDB fallback
packages/domain/        # Pure types, validators, selectors (zero dependencies)
packages/ui/            # Shared UI primitives (shadcn/ui, Radix, Tailwind v4)
```

## Required Environment Variables

Create `apps/web/.env` (not project root):

```
VITE_SUPABASE_URL=<your-supabase-project-url>
VITE_SUPABASE_PUBLISHABLE_DEFAULT_KEY=<your-supabase-anon-key>
```

CI generates `.env` from GitHub Actions variables (see `.github/workflows/deploy.yml`).

## Non-Obvious Gotchas

- **`routeTree.gen.ts`** is auto-generated by TanStack Router — do not edit manually.
- **Tailwind v4** has no config file; uses `@tailwindcss/vite` plugin. Import `tailwindcss` directly in CSS.
- **`pnpm typecheck` is not part of `pnpm dev`** — run separately before committing.
- **`pnpm test` is a placeholder** — no test framework installed despite README mentioning Vitest.
- **GitHub Pages builds** need `GITHUB_PAGES=true` env var (set automatically in CI).
- **Biome** is configured for linting and formatting (see `biome.json`). Run `pnpm lint` to check, `pnpm format` to auto-fix.
- **Supabase MCP** is configured in `.vscode/mcp.json` for remote schema access.

## Style

Follow `~/.claude/rules/typescript/coding-style.md` (interfaces > types for objects, no `React.FC`, explicit return types on exports, no `any`, no `console.log` in production code).
